### Stop running migrations and creating superusers at container boot (H8 + `DJANGO_SUPERUSER_*`)
The current `entrypoint.sh` still runs `manage.py migrate` on every boot (a race under multi-replica rolling update). Planned replacement:

1. Add a one-shot ECS task definition (`itg-backend-migrate`) that runs `manage.py migrate --noinput` and exits.
2. In `deploy-backend.yml`, run that task definition after the new image is pushed but **before** `amazon-ecs-deploy-task-definition@v2` updates the service.
3. Remove the `python manage.py migrate` line from `entrypoint.sh`.
4. Superuser creation (`DJANGO_SUPERUSER_*`) should become a manual, once-per-environment step, not a boot-time side-effect.

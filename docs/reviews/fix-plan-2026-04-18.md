## Execution order

1. Batches 1-2 first (critical security, no infra).
2. Batch 5 next (adds CI gates so future regressions are caught).
3. Batches 3, 4, 6, 7 in parallel-friendly order (independent files).
4. Batch 8 next (tests harden what we just fixed).
5. Batch 9 last (dep upgrades — highest risk of breakage).

Each batch ends with:

```bash
cd src && ruff check . && ruff format --check . && python manage.py test --settings=core.settings.dev
cd pages && npm run lint && npx tsc --noEmit && npm test && npm run build
```

…all green before committing.

Welcome to I2G

This system is primarily written in Python, JavaScript, HTML, and CSS. It is a Flask application, and the docs are linked below:

Flask docs: https://flask.palletsprojects.com/en/stable/

Running the system for the first time

After cloning the repository to your local machine, it is best to utilize a virtual environment to run the application locally.

To create a virtual environment on mac, run:

python3 -m venv .venv

This creates a virtual environment using Python's venv module called .venv.

To start the virtual environment, run:

source .venv/bin/activate

Now you must install all of the dependencies. Run:

python3 -m pip install -r requirements.txt

Now you can run the application:

python3 wsgi.py

Common Error

An error you might encounter is that the gspread config is in the wrong place. In the repository, it is located in:

.config/gspread/service_account.json

However, the file service_account.json should be in your home directory. Move the file to:

~/.config/gspread/service_account.json

This should resolve the error.

Important Files and Folders

In the main folder, wsgi.py is the entry point of the app.

The two important subdirectories are config and project:

  • config contains constants that need to be used throughout the application.
  • project contains the bulk of the code for the application. The rest of the documentation will go over what is in this folder.

The following files in the project directory are critical for the functioning of the I2G system:

  • admin.py
    • The admin dashboard for I2G.
    • Handles catching email bounces.
    • Allows creation and editing of events.
    • Manages admin users and roles.
  • event.py
    • Handles event registration for users already in the Google Sheet.
    • Updates and adds entries to the Google Sheet.
    • Sends confirmation emails to registered users.
  • registration.py
    • Handles event registration for new users (not already in the Google Sheet).
    • Updates and adds entries to the Google Sheet.
    • Sends confirmation emails to users registered for the event.

Common Tasks

Every semester, there is an I2G event. Therefore, there are routine tasks that must be completed before, during, and after each I2G event. Here is a list of common tasks and how to complete them:

  1. Change home page to home-pre-event.html
    1. Navigate to project/views/home.py and choose the correct template in the mainpage function.
  2. Change times in the home-pre-event.html and event.html
    1. These times must reflect the times of the upcoming event.
    2. Unfortunately, these times are hardcoded, so they must be changed manually in both HTML files.

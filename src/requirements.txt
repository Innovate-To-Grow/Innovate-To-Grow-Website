# Dependency manifest entrypoint.
#
# Dependencies are split under requirements/:
#   requirements/base.txt        — shared across all environments
#   requirements/production.txt  — base + WSGI/ASGI servers, PostgreSQL, Redis, S3
#   requirements/local.txt       — production + dev/CI tooling (ruff, ...)
#
# This file resolves to the local/CI closure so every existing consumer
# (`pip install -r requirements.txt`, the Docker build, pip-audit, and the
# setup-python pip cache keyed on src/requirements.txt) keeps working unchanged.
# For a lean production image, install requirements/production.txt instead.

> **Stacked PR** — base 是 `security/login-link-hardening`（#278）。请先审/合 #278，再看本 PR 的增量。

### Motivation

#278 把邮件自动登录链接统一收紧为短有效期 + 一次性，但保留了历史上的**两套平行机制**（campaign 的 `MagicLoginToken` DB 行 + event 自己的签名 ticket token），且一刀切的策略破坏了登录链接作为「个性化邮件 post-login 入口」的产品定位——票务登录本质上就是"目的地为票务页的登录链接"（现状甚至连目的地都没实现：`TicketAutoLoginView` 响应里没有 `redirect_to`，登录后落在 `/account`）。

本 PR 把两套机制统一为单一的 **`LoginLinkToken`** 原语，并把策略（有效期 / 可重用 / 撤销）下放给发件来源配置。

### Design

| 维度 | campaign 邮件（`{{login_link}}`） | 票务确认邮件 |
|---|---|---|
| 签发 | `runner._build_login_link` → `issue_login_link()` | `ticket_mail.send_ticket_email` → `issue_login_link()` |
| 登录后目的地 | `campaign.login_redirect_path`（实时读取） | token 行 `redirect_path` = `/event-registration` |
| 有效期（发送时冻结进 `expires_at`） | `EmailCampaign.login_link_validity_days`，默认 **7**，1–90，发送后锁定 | `Event.ticket_login_validity_days`，默认 **30**，1–90 |
| 可重用 | `EmailCampaign.login_link_reusable`，默认 **关** | `Event.ticket_login_reusable`，默认 **开** |
| 撤销 | campaign changelist action + Login Links admin | registration changelist action + Login Links admin |

**Kill switch**：可重用开关在登录时**实时读取**（不冻结在 token 上）。reusable 登录会记录 `is_used`/`used_at`（审计），所以事后取消勾选 → 已用过的链接立即失效、未用过的只剩一次。来源被删除（orphan token）安全降级为一次性。

**Admin**：新增只读 **Login Links** changelist（member / campaign / registration / 状态徽章 / revoke action，**raw token 永不展示**）；campaign 发送后锁定 validity 但保持 reusable 可编辑；revoke 权限走 `mail.change_loginlinktoken`。

### 前端行为改进

`LoginLinkPage` 在 token 被拒但本地已有会话时静默跳转 `/account`（一次性链接二次点击不再是死胡同）；该路径有 vitest + e2e 双覆盖。

### 多镜头对抗审查（22 agents）后的修复与取舍

审查确认 3 个问题并已修复：

1. **`is_active` 门禁回归（high）**——旧票务流拒绝停用账号、统一视图漏掉了 → `LoginLinkView` 现在以同样的泛化错误信息拒绝停用成员（不泄露账号状态），一次性与 reusable 路径都有测试。
2. **kill-switch 的过期边界 TOCTOU（low）**——`record_reusable_use()` 改为 `WHERE expires_at > now` 的条件 UPDATE 并返回布尔值；撤销与登录并发时不再可能"边撤边登"。
3. **admin 状态徽章误导（low）**——已用但仍可重用的链接现在显示 `reusable` 而非 `used`；`list_select_related` 补了 `registration__event` 防 N+1。

两条**有意的设计取舍**（审查标记、保留并记录）：

- **票务链接默认 30 天可重用**：这是本 PR 的核心产品决策（票务链接=post-login 入口），相比 #278 的 24h 一次性是放宽——但换来了 #278 之前没有的撤销能力、kill switch、重发即换链，且组织者可按活动调整。
- **跨成员 fallback**：dead link + 浏览器已有他人会话 → 落在该会话的 /account。与直接访问 /account 等价，失败的 token 不泄露属主信息；已在代码注释说明。
